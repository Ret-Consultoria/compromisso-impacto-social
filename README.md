### 1. 🔓 Open Source — Código Aberto, Impacto Aberto

Seguimos o modelo de **OSPO (Open Source Program Office)** adotado por Google, Microsoft e Red Hat: contribuições sistemáticas, não aleatórias.

| Princípio | Prática | Referência |
|-----------|---------|-----------| 
| 🌐 **Acessibilidade** | Ferramentas gratuitas que nivelam o campo | Mozilla, Automattic |
| 🤝 **Colaboração** | Contribuições em projetos da comunidade global | IBM, Shopify |
| 🔍 **Transparência** | Código público, auditável, sob licença MIT | Linux Foundation |

<br/>

- ✅ Relatórios profissionais (PDF, JSON, Markdown) para entrega a clientes
- ✅ Licença **MIT — 100% livre**, sem vendor lock-in
- ✅ Fortalece a segurança do ecossistema digital brasileiro

## 🤝 Como Contribuir

Contribuições são bem-vindas! Quer adicionar uma frente de impacto, sugerir parcerias ou melhorar documentação:

1. **Fork** este repositório
2. Crie uma **branch** com sua feature (`git checkout -b feature/nova-frente`)
3. **Commit** suas mudanças (`git commit -m 'feat: adiciona parceria com ONG X'`)
4. **Push** para a branch (`git push origin feature/nova-frente`)
5. Abra um **Pull Request**

> 💡 **Dica:** Siga [Conventional Commits](https://www.conventionalcommits.org/) para mensagens de commit.
